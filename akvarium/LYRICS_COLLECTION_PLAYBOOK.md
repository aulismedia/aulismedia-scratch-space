# Playbook: Сбор полного корпуса текстов песен исполнителя

## Цель

Собрать все тексты песен исполнителя/группы в единый структурированный JSON для последующего анализа (NLP, статистика, визуализация, обучение моделей и т.д.).

Итоговая структура:
```json
[
  {
    "album": "Название альбома",
    "year": 1987,
    "source": "merged",
    "songs": [
      { "title": "Название песни", "lyrics": "Текст песни" }
    ]
  }
]
```

---

## Принципы

### 1. Несколько независимых источников

Никогда не полагайся на один источник — каждый имеет пробелы:
- **Аккордовые сайты** (accords.site, amdm.ru, ultimate-guitar.com): широкое покрытие, но тексты смешаны с аккордами.
- **FB2/epub-библиотеки** (flibusta, lib.ru): архивные записи, часто охватывают ранний период.
- **Официальный сайт исполнителя**: самый авторитетный источник, чистые тексты, но неполное покрытие по старым релизам.
- **Стриминги + Bandcamp**: иногда содержат тексты в метаданных (проверять `data-tralbum` JSON).
- **Фан-сайты и вики**: могут закрывать пробелы для редких треков.

Источники обрабатываются независимо → потом мёрджатся с дедупликацией.

### 2. Иерархия авторитетности источников

```
Официальный сайт > аккордовый сайт > FB2/epub > фан-сайты
```

При конфликте версий текста: предпочитать официальный сайт. При конфликте метаданных (название альбома, год): предпочитать официальный сайт или Discogs.

### 3. Что считать официальным релизом

Перед сбором определить границы датасета. Типичные категории:

| Тип | Включать? |
|-----|-----------|
| Студийные альбомы | Да |
| Концертные/live | На усмотрение |
| Саундтреки | На усмотрение |
| Каверы | На усмотрение |
| Сборники чужих исполнений другим автором | **Нет** (чужой материал) |
| Бутлеги (только в неофициальных источниках) | Нет |
| Синглы / EP | На усмотрение |

Бутлеги детектируются по признаку: альбом присутствует **только** в неофициальных источниках (FB2, фан-сайты) и отсутствует в аккордовых сайтах и официальных источниках.

## Дедупликация альбомов

Одинаковые альбомы в разных источниках имеют разные названия — нужна нечёткая сверка.

**Нормализация перед сравнением:**
- ё → е
- Удалить `(YEAR)` из названия
- Удалить `Том N`, `Vol. N`
- Всё в lowercase, убрать знаки препинания

**Сравнение:** `difflib.SequenceMatcher` с порогом ≈ 0.82

**Спецкейсы, требующие ручных правил:**
- Год как часть названия («Боб — 87» vs «Боб — 88»): если оба имени содержат разные двузначные числа → сходство = 0, не мёрджить.
- Prefix-match бонус: если короткое название является префиксом длинного («Чёрная Роза» vs «Чёрная Роза — эмблема печали...») → бонус +0.35.

**Мёрдж песен внутри альбома:** по нормализованному названию (ё→е, убрать скобки, lowercase). При конфликте — оставить версию с большим текстом.

## Очистка текстов от аккордов

Аккордовые сайты вставляют аккорды в текст. Алгоритм очистки построчный:

### Строки для удаления целиком
- **Диаграммы** (`+-+-+-+`, `| | | 0 | |`, цифровые fret-паттерны)
- **Строки только с аккордами** (`Am G C7 F#m`, `G7sus4 Hm F#`)
- **Гитарный таб** (`e|--0==---`, `e[6(0)],g[6(3)]`)
- **Аккорды с римскими цифрами** (`Am(V)`, `G(III),Em(II),D,Hm`)
- **Нотная мелодия** (`a - - - g f e - - d`)
- **Email/attribution** (`From: user@mail.ru`)
- **Структурные маркеры** (`instr:`, `[Instr]`, `Bonus tracks:`)

### Кириллические строки — удалить «шум» с краёв
- Ведущие аккорды перед первой кириллической буквой
- Trailing аккорды/диаграммы после последней кириллической буквы с пунктуацией

### Ключевая эвристика
- **Есть кириллица** → строка текста (зачистить края)
- **Нет кириллики, есть длинные (5+ букв) строчные английские слова** → английские lyrics (оставить)
- **Нет кириллики, нет длинных английских слов** → аккорды/шум (удалить)

### Расширенный `CHORD_TOKEN` regex (покрывает большинство случаев)
```python
CHORD_TOKEN = (
    r"[A-H][#b]?"
    r"(?:m(?:aj|in)?|dim|aug|sus|add)?"
    r"\d*"
    r"(?:sus[24])?"
    r"(?:[/\\][A-Ha-h][#b]?)?"   # bass note — допускать строчные (D/f#)
    r"(?:\([IVX]+\))?"            # fret position (III), (V)
)
```

## Официальный сайт исполнителя как источник чистых текстов

Если у исполнителя есть официальный сайт с разделом lyrics — он **приоритетен**:
- Тексты уже без аккордов
- Правильные названия и орфография

**Типичная структура (Drupal/WordPress):**
```
/ru/music           — список альбомов
/ru/album/<slug>    — страница альбома → ссылки на треки
/ru/lyrics/<slug>   — страница трека → чистый текст
```

**Извлечение текста:** искать `<article>` внутри `div.layout-content` (или аналогичного контентного контейнера). Не брать текст через `h1.next_siblings` — в Drupal h1 и контент часто в разных ветках DOM.

## Типичные проблемы и решения

| Проблема | Решение |
|----------|---------|
| SSL-сертификат истёк на сайте | Использовать HTTP вместо HTTPS |
| Контент загружается через JS (AMDM) | Найти альтернативный источник, не пытаться selenium без крайней необходимости |
| FB2 с битым XML | `lxml.etree.XMLParser(recover=True)` |
| FB2 с плоской структурой (всё в одном `<body>`) | Парсить через state machine по строкам |
| Название альбома на официальном сайте ≠ название в датасете | Вести явный маппинг `ALBUM_NAME_MAP` в скрипте добавления |
| Один трек под двумя разными названиями в разных источниках | Сравнивать первые ~80 символов текста; при совпадении — оставить версию с большим текстом, обновить название |
| Сингл/новый трек без published lyrics | Зафиксировать, добавить позже вручную |

## Экспорт в TXT

Финальный JSON можно конвертировать в читаемый текстовый файл (для просмотра, diff-а, поиска).

### Формат

```
============================================================
Альбом "Название альбома" издан в YYYY году
============================================================

1. Название песни

Текст первого куплета
...

----------------------------------------

2. Название песни

Инструментал

----------------------------------------
```

- Разделитель альбома: `=` × 60
- Разделитель песни: `-` × 40
- Инструментал / нет текста: строка `Инструментал`

### Скрипт `07_export_txt.py`

```python
"""
Step 7 (utility): Export artist_lyrics.json → artist_lyrics.txt
"""

import json
from pathlib import Path

BASE = Path(__file__).parent
INPUT  = BASE / "artist_lyrics.json"     # поменять при необходимости
OUTPUT = BASE / "artist_lyrics.txt"

SEP_ALBUM = "=" * 60
SEP_SONG  = "-" * 40

def main():
    data = json.loads(INPUT.read_text())
    lines = []

    for album in data:
        name = album.get("album", "Без названия")
        year = album.get("year")
        year_str = f" в {year} году" if year else ""
        lines += [SEP_ALBUM, f'Альбом "{name}" издан{year_str}', SEP_ALBUM, ""]

        for i, song in enumerate(album.get("songs", []), 1):
            title   = song.get("title", "Без названия")
            lyrics  = (song.get("lyrics") or "").strip()

            lines.append(f"{i}. {title}")
            lines.append("")
            lines += (lyrics.splitlines() if lyrics else ["Инструментал"])
            lines += ["", SEP_SONG, ""]

    OUTPUT.write_text("\n".join(lines), encoding="utf-8")
    print(f"Saved → {OUTPUT}")

if __name__ == "__main__":
    main()
```

### Запуск

```bash
python3 07_export_txt.py
```

## Чеклист качества перед завершением

- [ ] Все альбомы из официальной дискографии присутствуют
- [ ] Нет альбомов-дубликатов (проверить похожие названия)
- [ ] Нет песен с текстом «Поиск по сайту» или другим nav-мусором
- [ ] Нет песен с 0 символами текста (кроме инструменталов)
- [ ] Нет строк только с аккордами в текстах (`grep` по `^[A-H][#b]?m?\s`)
- [ ] Нет email/attribution строк (`grep` по `From:`, `@`)
- [ ] Нет строк `[Instr]`, `Bonus tracks:`, `instr:`
- [ ] Английские тексты (для билингвальных исполнителей) сохранены корректно
- [ ] Год выпуска заполнен для всех альбомов (кроме сборников без даты)
